# Production Script — Governed Decision Intelligence
## Abraham of London
### Full Kling-Optimised Video Script — 3 minutes 20 seconds

---

## Production Notes for Kling

**Kling 2.0 workflow:**
- Generate each scene as a separate 10-15 second Kling clip
- Use Image-to-Video for product surface mockups (upload static designs, animate them)
- Use Text-to-Video for abstract visuals (decision maps, corridors, gates)
- Stitch in post-production with consistent grade
- Max clip length: 2 minutes (Pro) — but 10-15 second clips are sharper

**Visual style tokens for Kling prompts:**
- `cinematic, dark institutional, charcoal black background, deep navy, muted gold accents`
- `slow camera movement, architectural precision, no people, no stock photography`
- `serif typography, clean lines, restrained motion, premium minimalism`
- `film grain, shallow depth of field, volumetric lighting`

**Audio:** Voiceover recorded separately (ElevenLabs or professional VO). Music: low cinematic tension, no corporate uplift.

## Scene 5 — The Intelligence Engine (25 seconds) ★ MASTERPIECE MOMENT

**Kling prompt:**
```
Dark screen. A cursor blinks. Text appears as if someone is typing: "I need to grow revenue." A system response appears below: "That is an outcome, not a decision." The text revises to: "I need to decide whether to acquire or build." A signal band animates across the screen with three metrics: EVIDENCE, AUTHORITY, CONSEQUENCE. Each fills to a different level. Cinematic, precise, typographic. 4K.
```

**On-screen text (appears as interaction):**
```
User: "I need to grow revenue."
System: "That is an outcome, not a decision."
User: "I need to decide whether to acquire or build."
System: "Decision condition identified. Testing evidence..."
[Signal band appears]
Evidence: MODERATE
Authority: UNCLEAR
Consequence: STRUCTURAL
```

**Voiceover:**
The system does five things.
First, it names the condition. Not vague risk. Not generic misalignment.
A real decision pattern, supported by evidence.
Second, it tests the evidence. If the record is too thin, the system says so.
Third, it shows consequence. Not as certainty. As a governed estimate, source-labelled and caveated.
Fourth, it controls progression. Users do not click their way into deeper surfaces. They earn progression through evidence.
Fifth, it verifies outcome. The system returns to ask what happened, whether the correction was taken, whether the signal reduced.
That is where intelligence becomes institutional memory.

**Duration:** 25 seconds
**Kling clip:** 1 × 15s (typing interaction) + 1 × 10s (signal band)

### Masterpiece Moments (4 scenes that prove excellence)

These four scenes are where the viewer *sees* the intelligence, not just hears about it:

1. **Scene 5** — The Intelligence Engine: User types, system parses, signal band appears. Proves the system thinks.
2. **Scene 6** — The Refusal Engine: Gate evaluates and refuses. Proves the system has standards.
3. **Scene 8** — The Compounding Record: Case record grows over time. Proves the system remembers.
4. **Scene 9** — The Evidence Detail: Full provenance on a single record. Proves the system is engineered, not marketed.

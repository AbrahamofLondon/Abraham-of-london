# Reporting Output Layer — Methodology Boundary

**Document ID:** REPORTING_OUTPUT_LAYER_METHODOLOGY  
**Effective Date:** 2026-06-14  
**Status:** Evidence Package Component  

---

## What This Layer Is

A unified governance framework for report generation across multiple product families. Reports are delivery artifacts that describe findings, interpret evidence, and support decision-making — but do not grant authority or validate claims.

The layer applies to:
- Monthly reports
- Custom reports  
- Quarterly intelligence reports
- Diagnostic reports
- Assessment reports
- Retainer oversight briefs
- Board-facing drafts

---

## What This Layer Is NOT

❌ An authority-granting mechanism  
❌ A claim-validation system  
❌ A certification process  
❌ An independent verification system  
❌ Investment, legal, or medical advice  
❌ A guarantee of accuracy or outcomes  

---

## Core Doctrine

**Reports are delivery artifacts, not authority-granting evidence artifacts.**

A report may:
- ✓ Describe evidence
- ✓ Interpret findings
- ✓ Package analysis
- ✓ Disclose source boundaries
- ✓ State limitations
- ✓ Support decision-making

A report must not:
- ✗ Become authority evidence by itself
- ✗ Restore product authority
- ✗ Validate claims
- ✗ Certify findings
- ✗ Provide investment/legal/medical advice

## Report Type Profiles

### Monthly Report
**Purpose:** Routine periodic reporting of findings and metrics  
**Boundary:** Input sources declared, human-reviewed  
**Allowed Claims:** Structured summary, decision-support material  
**Forbidden Claims:** Certified, validated, guaranteed  

### Custom Report
**Purpose:** On-demand analysis for specific scope  
**Boundary:** Evidence-limited advisory review  
**Allowed Claims:** Analysis of available evidence, advisory findings  
**Forbidden Claims:** Prediction certainty, external validation, authority  

### Quarterly Intelligence Report
**Purpose:** Evidence-limited market or industry analysis with prior-quarter review  
**Boundary:** Evidence-limited with falsification register  
**Allowed Claims:** Evidence-limited intelligence, structured analysis  
**Forbidden Claims:** Investment advice, prediction guarantees, AI forecasting  

### Diagnostic Report
**Purpose:** Decision-support analysis for specific decision point  
**Boundary:** Input sources declared, human-reviewed  
**Allowed Claims:** Decision-support material, evidence-based analysis  
**Forbidden Claims:** Guaranteed correctness, certified diagnosis  

### Assessment Report
**Purpose:** Evaluation of specific domain or capability  
**Boundary:** Evidence-limited with methodology disclosed  
**Allowed Claims:** Assessment summary, findings interpretation  
**Forbidden Claims:** Certified assessment, independently validated  

### Retainer Oversight Brief
**Purpose:** Ongoing monitoring and briefing under retainer  
**Boundary:** Input sources declared, human-reviewed  
**Allowed Claims:** Oversight findings, activity summary  
**Forbidden Claims:** Audit opinion, assurance opinion, guarantees  

### Board-Facing Draft
**Purpose:** Board discussion material for decision-support  
**Boundary:** Authority referenced but not granted  
**Allowed Claims:** Discussion material, analysis for board review  
**Forbidden Claims:** Board approval, board certification, external proof

## Required Declarations

Every report must include:

1. **Evidence Boundary Statement** (first section, visible)
   - Clear non-authority declaration
   - Scope and limitations
   - What the report is/is not

2. **Human Review Attestation**
   - Reviewer name and date
   - Confirmation that review was completed
   - Claim verification completed

3. **Input Source Declaration**
   - Sources used in analysis
   - Methodology overview
   - Known limitations

4. **Forbidden Claims Check**
   - Verification that no forbidden language present
   - If detected, claims removed before publication
   - Attestation from reviewer

5. **Confidence Language Boundaries**
   - Approved: "may indicate", "suggests", "consistent with"
   - Forbidden: "will happen", "guarantees", "predicts"

6. **Non-Authority Disclaimer**
   - Explicit statement that report does not grant authority
   - Confirmation report is delivery artifact
   - Statement that report may interpret but does not validate

## Input Source Rules

### Allowed Sources
- Institutional sources (IMF, central banks, policy bodies)
- Primary data (market data, published statistics, official documents)
- Market-implied signals (observable from public markets)
- Published research (peer-reviewed, investment banks, think tanks)
- Policy documents (official announcements, regulations)
- Internal analysis (product team findings, proprietary methodology)
- Expert review (qualified professional assessment)

### Forbidden Sources
- Proprietary hedge fund data
- Confidential client data
- Insider trading signals
- Unpublished government intelligence
- Undisclosed sources

---

## Human Review Requirements

All reports must pass human review including:

**Content Review:**
- [ ] Claim language is bounded (no certainty claims)
- [ ] Evidence sources are declared
- [ ] Boundary statement is complete and accurate
- [ ] Forbidden claims are absent
- [ ] Confidence language is appropriate
- [ ] Methodology is disclosed

**Authority Review:**
- [ ] Report does not restore authority
- [ ] Report does not grant certification
- [ ] Report does not claim external validation
- [ ] Report does not provide prohibited advice
- [ ] Report does not guarantee outcomes
- [ ] Evidence boundary statement is present

**Source Review:**
- [ ] Sources are declared
- [ ] Included sources are allowed types
- [ ] Excluded sources (if any) are documented
- [ ] Methodology is transparent
- [ ] Limitations are stated

## Report-As-Evidence Prohibition

**Fundamental Rule:** Reports do not become authority evidence.

**Implications:**
- Reports may NOT be cited as independent proof
- Reports may NOT restore product authority
- Reports may NOT validate claims made in other products
- Reports may NOT replace actual evidence artifacts
- Reports must always declare non-authority status

**Enforcement:**
- Every report includes non-authority disclaimer
- Reports explicitly state they are delivery artifacts
- Governance guard scans for violations
- Authority gates reject any authority claims from reports
- Violation triggers report revocation

---

## Revocation Conditions

A report is revoked and republished if:

1. **Report cited as authority evidence** — If used to prove a claim externally
2. **Forbidden claims introduced** — If language is found to violate boundaries
3. **Human review bypassed** — If published without required review
4. **Sources undeclared** — If methodology becomes hidden
5. **Authority restoration claimed** — If used to upgrade product state

---

## Enforcement

The governance guard verifies:
- No forbidden claims in report content
- Evidence boundary statement present
- Human review attestation present
- Source declaration present
- Non-authority disclaimer present
- Report not cited as authority evidence

Violation = report revocation and rewrite required before republication

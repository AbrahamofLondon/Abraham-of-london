# DECISION AUTHORITY MATRIX

**(A+ Strategic Framework — Core System Anchor)**

---

## 1. Executive Summary

Every organisation has decisions.
Few have **decision authority clarity**.

The Decision Authority Matrix defines:

```txt
Who decides
Who contributes
Who executes
Who is accountable
```

Without this:

```txt
Decisions stall  
Accountability disappears  
Execution fragments  
```

---

## 2. Strategic Context

Most organisations confuse:

```txt
Participation → ownership  
Influence → authority  
Responsibility → accountability  
```

This creates:

```txt
Decision ambiguity
Escalation loops
Leadership conflict
```

---

## 3. Core Model

Each decision is mapped across **4 roles**:

```txt
A — Authority (final decision maker)
C — Contributor (input provider)
E — Executor (implements decision)
R — Responsible (owns outcome)
```

---

## 4. Matrix Structure

Example:

```txt
Decision: Market Expansion

Authority     → CEO
Contributor   → CFO, Strategy Lead
Executor      → Operations
Responsible   → Regional Director
```

---

## 5. Failure Conditions

```txt
Multiple authorities → conflict
No authority → paralysis
Contributor acting as authority → shadow power
Executor without clarity → failure
```

---

## 6. Diagnostic Signals

```txt
AUTHORITY_VACUUM
ESCALATION_OVERLOAD
DECISION_LATENCY
TRUST_ASYMMETRY
```

---

## 7. Operating Logic

```txt
Every decision must have exactly 1 Authority
Authority cannot be shared
Responsibility must align with execution
Contributors cannot override Authority
```

## 8. Application Playbook

### Step 1 — List key decisions

Top 10 strategic + operational

### Step 2 — Assign roles

A, C, E, R for each

### Step 3 — Validate

```txt
No duplicates in Authority
No gaps in Responsibility
No overlap confusion
```

### Step 4 — Implement

* communicate roles
* enforce decision rules
* track delays

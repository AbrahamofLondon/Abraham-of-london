## 1. Executive Summary

Most organisations do not fail at strategy.
They fail at execution.

Not because they lack capability—but because:

```txt
Decisions are not enforced  
Ownership is unclear  
Failure is tolerated  
Performance is misreported  
```

The Execution Integrity Grid identifies:

```txt
Where execution breaks  
Why it breaks  
Who is responsible  
What must be enforced  
```

It converts execution from:

```txt
Effort → Outcome  
```

---

## 2. Strategic Context

Execution sits between:

```txt
Decision → Outcome
```

Most organisations assume this link is automatic.

It is not.

Breakdowns occur across four dimensions:

```txt
Clarity     — Was the decision understood?
Ownership   — Is someone accountable?
Capability  — Can it be executed?
Enforcement — Is failure corrected?
```

If any dimension fails → execution collapses.

---

## 3. Core Model

The grid operates as a **4×4 diagnostic system**:

```txt
                LOW INTEGRITY        HIGH INTEGRITY
---------------------------------------------------------
CLARITY         Confusion            Clear direction
OWNERSHIP       Diffused             Single owner
CAPABILITY      Inadequate           Fit for purpose
ENFORCEMENT     Ignored              Actively enforced
```

Each dimension is scored independently.

---

## 4. Scoring Model

```txt
0–40     = FAILURE
41–60    = WEAK
61–80    = FUNCTIONAL
81–100   = STRONG
```

Composite output determines execution state.

---

## 5. Execution States

```txt
ORDERED        — Execution consistent and predictable
FRAGILE        — Works under normal conditions only
DEGRADING      — Performance declining
BROKEN         — System not delivering outcomes
```

## 7. Failure Modes

### 7.1 Clarity Failure

```txt
Decision unclear
Instructions inconsistent
Teams interpret differently
```

### 7.2 Ownership Failure

```txt
No accountable individual
Shared responsibility
Escalation without resolution
```

### 7.3 Capability Failure

```txt
Team lacks skill
System cannot support execution
Process too complex
```

### 7.4 Enforcement Failure

```txt
Failure tolerated
Deadlines ignored
No consequence for non-performance
```

---

## 8. Operating Logic

Execution integrity requires:

```txt
One clear decision  
One accountable owner  
Defined success metric  
Enforced deadline  
```

If any element is missing → execution is invalid.

### Step 4 — Classify system state

```txt
Ordered / Fragile / Degrading / Broken
```

### Step 5 — Apply correction

Target weakest dimension first

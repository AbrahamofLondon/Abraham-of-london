# INSTITUTIONAL DIAGNOSTICS — INTERVENTION MATRIX

**(A+ Standard — Diagnostic-to-Intervention Mapping System)**

---

## 1. Executive Summary

Diagnosis without intervention is observation.

Most institutional diagnostics produce accurate findings and then fail at the critical moment — the transition from identifying the problem to solving it. The failure is not in the diagnosis. It is in the absence of a structured intervention selection mechanism.

This matrix provides:

```txt
Finding-to-Intervention Mapping — every diagnostic outcome has a prescribed intervention path
Intervention Sequencing — the correct order of operations for multi-dimensional failure
Resource Allocation Rules — how to deploy limited intervention capacity for maximum effect
Success Criteria — how to know when an intervention has worked
```

It eliminates the gap between knowing what is wrong and doing what is required.

---

## 2. Strategic Context

Interventions fail in five predictable patterns:

```txt
1. Wrong intervention — diagnosis correct, treatment wrong. Stabilisation applied where restructuring was required.

2. Right intervention, wrong sequence — all the right actions, in the wrong order. Governance reform before authority clarity. Culture change before structural change.

3. Right intervention, insufficient intensity — correct action, inadequate resources or duration. The problem outlasts the intervention.

4. Right intervention, wrong owner — correct action assigned to someone without authority to execute it.

5. No intervention — diagnosis complete, no action taken. Analysis paralysis.
```

### The Intervention Cost Curve

```txt
Early intervention (signal detected at LOW-MEDIUM):
  Cost: Low
  Duration: Weeks
  Success Rate: >80%

Late intervention (signal detected at HIGH):
  Cost: Moderate-High
  Duration: Months
  Success Rate: 40-60%

Crisis intervention (signal detected at CRITICAL):
  Cost: High-Very High
  Duration: Months-Years
  Success Rate: 20-40%
```

The earlier the intervention, the lower the cost and the higher the success rate.

## 3. Core Model — Intervention Matrix

### 3.1 The Five Intervention Classes

```txt
Class 1 — Corrective
  Purpose: Fix a specific identified failure
  Scope: Localised, single domain
  Duration: 30 days
  Example: Replace a failing process owner

Class 2 — Structural
  Purpose: Redesign a failing system or process
  Scope: Distributed, single domain
  Duration: 60 days
  Example: Redesign the decision rights matrix

Class 3 — Cultural
  Purpose: Realign behaviour with declared values
  Scope: Distributed, cross-domain
  Duration: 90 days
  Example: Install consequence system for cultural standards

Class 4 — Transformational
  Purpose: Restructure institutional architecture
  Scope: Systemic, all domains
  Duration: 6-12 months
  Example: Full governance redesign

Class 5 — Existential
  Purpose: Prevent institutional collapse
  Scope: Systemic, immediate
  Duration: Ongoing until stabilised
  Example: Crisis command activation
```

### 3.3 Intervention Sequencing Rules

```txt
Rule 1: Authority before accountability
  — Clarify who decides before assigning who answers
  — Sequence: Mandate Clarity → Accountability Framework

Rule 2: Structure before culture
  — Fix the system before asking behaviour to change
  — Sequence: Governance Redesign → Culture Intervention

Rule 3: Containment before correction
  — Stop the bleeding before designing the cure
  — Sequence: Crisis Protocol → Root Cause Analysis → Structural Fix

Rule 4: Leadership before team
  — Address leadership gaps before team-level intervention
  — Sequence: Leadership Formation → Team Alignment

Rule 5: One intervention at a time
  — Do not launch multiple Class 3+ interventions simultaneously
  — Maximum: 1 transformational + 2 structural + unlimited corrective
```

## 4. Resource Allocation Rules

### 4.1 Intervention Capacity Model

```txt
Each institution has finite intervention capacity:

Capacity = Available leadership attention × Execution bandwidth × Financial resources

Rule: Never commit >60% of intervention capacity to a single domain.
Rule: Always reserve 20% capacity for emerging issues.
Rule: Never launch a Class 4 intervention without dedicated programme management.
```

### 4.2 ROI-Based Prioritisation

```txt
Priority 1: Interventions that remove existential threat (Class 5)
Priority 2: Interventions that unblock other interventions (sequencing rules)
Priority 3: Interventions with highest impact-to-effort ratio
Priority 4: Interventions that build institutional capacity for future interventions
```

## 5. Success Criteria

### 5.1 Intervention Completion Criteria

```txt
An intervention is complete when:

1. The specific failure identified in the diagnosis is resolved
2. The fix has survived one full operating cycle without regression
3. The owner has been transitioned from intervention team to business-as-usual
4. Documentation has been updated to reflect the new state
5. Lessons learned have been captured and shared
```

### 5.2 Intervention Failure Criteria

```txt
An intervention has failed when:

1. The same failure recurs within two operating cycles
2. The intervention created new failures in adjacent systems
3. The intervention was not completed within 1.5x the planned timeline
4. The intervention required repeated escalation to sustain momentum
5. The intervention was reversed within one operating cycle of completion
```

## 6. Diagnostic Signals

Signals that the intervention matrix is needed or failing:

```txt
DIAGNOSIS_COMPLETE_NO_ACTION  — diagnostic completed but no intervention selected
INTERVENTION_MISMATCH         — intervention does not match diagnostic finding
INTERVENTION_SEQUENCE_ERROR   — interventions applied in wrong order
INTERVENTION_CAPACITY_EXCEEDED — too many interventions launched simultaneously
INTERVENTION_FATIGUE          — organisation exhausted by continuous change
INTERVENTION_REVERSAL         — intervention outcomes reversed after completion
```

---

## 7. Failure Modes

```txt
1. Analysis paralysis — diagnostic completed, intervention never selected. The organisation knows what is wrong and does nothing.

2. Scattershot intervention — multiple small interventions across multiple domains. None has sufficient intensity to succeed.

3. Premature escalation — Class 4 intervention applied to a Class 1 problem. Over-resourced and over-engineered.

4. Insufficient intervention — Class 1 corrective applied to a Class 3 cultural problem. Under-resourced and ineffective.

5. Owner mismatch — intervention assigned to someone without authority to execute it. Stalls at first resistance.

6. Intervention fatigue — organisation subjected to continuous change without recovery periods. Change resistance builds.
```

## 8. Application Playbook

### Step 1 — Map Diagnostic Findings

List all findings from the Institutional Diagnostics Toolkit or equivalent assessment.

**Deliverable:** Diagnostic Findings Register

### Step 2 — Select Intervention Class

For each finding, use the Finding-to-Intervention Mapping to select the correct intervention class.

**Deliverable:** Intervention Selection Matrix

### Step 3 — Apply Sequencing Rules

Check sequencing rules. Ensure interventions are ordered correctly.

**Deliverable:** Intervention Sequence Plan

### Step 4 — Allocate Resources

Apply resource allocation rules. Ensure capacity is not exceeded.

**Deliverable:** Resource Allocation Plan

### Step 5 — Assign Owners

For each intervention, assign an owner with authority to execute.

**Deliverable:** Intervention Ownership Matrix

### Step 6 — Install Success Criteria

Define completion and failure criteria for each intervention.

**Deliverable:** Intervention Success Criteria Document

### Step 7 — Monitor and Adjust

Review intervention progress at each governance cycle. Adjust as needed.

**Deliverable:** Intervention Progress Dashboard

## 9. Board-Level Questions

```txt
Do our interventions match our diagnostic findings — or are we treating symptoms?

Are our interventions in the right sequence — or are we creating new problems?

Do we have sufficient intervention capacity — or are we over-committed?

Who owns each intervention — and do they have the authority to execute?

How do we know when an intervention has succeeded — and when it has failed?

What is the cost of not intervening — and have we priced it?
```

## 10. Metrics & Measurement

```txt
Intervention Match Rate     — % of interventions that match diagnostic finding (target: >90%)
Intervention Completion Rate — % of interventions completed within 1.5x timeline (target: >80%)
Intervention Success Rate   — % of interventions meeting success criteria (target: >70%)
Intervention Reversal Rate  — % of interventions reversed within one cycle (target: <10%)
Sequence Compliance         — % of interventions following sequencing rules (target: 100%)
Time from Diagnosis to Intervention — days (target: <30 for Class 1-2, <60 for Class 3+)
```

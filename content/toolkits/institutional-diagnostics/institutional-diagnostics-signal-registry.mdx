# INSTITUTIONAL DIAGNOSTICS — SIGNAL REGISTRY

**(A+ Standard — Institutional Signal Classification System)**

---

## 1. Executive Summary

Every institutional failure sends signals before it arrives.

The signals are visible — to someone, somewhere in the organisation — long before the failure becomes undeniable. The problem is not that the signals do not exist. It is that they are not recognised, classified, or escalated.

This registry provides:

```txt
Complete signal taxonomy — 48 signals across 6 institutional domains
Severity classification — how to distinguish noise from warning from critical
Escalation thresholds — when a signal requires action, not just attention
Signal interaction mapping — how signals combine to indicate systemic failure
```

It transforms institutional monitoring from reactive (responding to failure) to predictive (detecting failure before it arrives).

---

## 2. Strategic Context

Signals fail in four ways:

```txt
1. Not detected — signal exists but no one is monitoring for it
2. Not classified — signal detected but severity not understood
3. Not escalated — signal classified but no action taken
4. Not acted on — signal escalated but ignored due to competing priorities

Each failure point has a structural correction. This registry addresses all four.
```

### The Signal-to-Failure Timeline

```txt
T-6 months: Weak signals detectable by systematic monitoring
T-3 months: Clear signals visible to those paying attention
T-1 month: Strong signals visible to most observers
T-0: Failure event

Most organisations detect signals at T-1 month.
Effective institutions detect signals at T-6 months.
```

## 3. Core Model — Signal Taxonomy

### 3.1 The Six Signal Domains

```txt
Domain 1 — Governance Signals      (authority, accountability, decision-making)
Domain 2 — Cultural Signals         (behaviour, norms, values alignment)
Domain 3 — Operational Signals      (execution, process, system performance)
Domain 4 — Financial Signals        (revenue, cost, cash flow, margin)
Domain 5 — Strategic Signals        (positioning, competitive, market)
Domain 6 — Human System Signals     (talent, leadership, team health)
```

### 3.2 Signal Classification

Each signal is classified across four dimensions:

```txt
Type:         Leading (predictive) / Lagging (confirmatory)
Severity:     Low / Medium / High / Critical
Confidence:   Speculative / Plausible / Likely / Confirmed
Scope:        Localised / Distributed / Systemic
```

### 3.3 Severity Definitions

```txt
LOW       — Isolated issue. No immediate impact. Monitor.
MEDIUM    — Repeating pattern. Moderate impact. Investigate.
HIGH      — System-level failure. Significant impact. Act.
CRITICAL  — Institutional threat. Severe impact. Escalate immediately.
```

## 5. Signal Interaction Mapping

### 5.1 Signal Combinations That Indicate Systemic Failure

```txt
GOV-02 + CUL-02 + OPS-01 = Governance-Culture-Operations cascade
  — Authority ambiguity + misaligned rewards + execution friction
  — Indicates: Systemic misalignment across leadership, culture, and operations
  — Required response: Institutional Diagnostics Toolkit — full assessment

FIN-03 + STR-05 + HUM-01 = Revenue-Talent-Culture cascade
  — Revenue concentration + talent decline + voluntary attrition
  — Indicates: Structural vulnerability masked by current revenue
  — Required response: Strategic repositioning + talent strategy overhaul

GOV-05 + CUL-05 + OPS-02 = Accountability-Execution cascade
  — Accountability vacuum + blame dispersal + error recurrence
  — Indicates: No one owns failure, so failure repeats
  — Required response: Accountability framework installation
```

### 5.2 Signal Velocity

```txt
The speed at which a signal moves from LOW to CRITICAL determines the response urgency:

Slow burn (6-18 months): Governance drift, cultural erosion, strategic fatigue
  → Response: Structured intervention, quarterly monitoring

Accelerating (3-6 months): Revenue concentration, talent decline, margin compression
  → Response: Active intervention, monthly monitoring

Rapid (1-3 months): Decision latency, escalation frequency, error recurrence
  → Response: Immediate intervention, weekly monitoring

Critical (<1 month): Any CRITICAL severity signal
  → Response: Escalate to Strategy Room
```

## 6. Escalation Protocol

### 6.1 Escalation Thresholds

```txt
LOW signal:
  — Log in signal register
  — Review at next monitoring cycle
  — No action required

MEDIUM signal:
  — Log in signal register
  — Assign owner for investigation
  — Report at next governance meeting
  — Action: Root cause analysis within 30 days

HIGH signal:
  — Escalate to senior leadership
  — Assign owner for resolution
  — Report at next board meeting
  — Action: Intervention plan within 14 days

CRITICAL signal:
  — Escalate immediately to board
  — Activate crisis protocol if applicable
  — Action: Intervention within 48 hours
```

### 6.2 Signal Review Cadence

```txt
Weekly:   Operational signals scan (OPS domain)
Monthly:  Governance + Financial signal review (GOV + FIN domains)
Quarterly: Full signal registry review (all domains)
Annually:  Signal registry calibration (are we monitoring the right signals?)
```

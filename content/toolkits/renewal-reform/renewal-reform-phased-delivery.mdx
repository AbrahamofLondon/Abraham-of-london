# RENEWAL & REFORM — PHASED DELIVERY SYSTEM

**(A+ Standard — Institutional Reform Execution Model)**

---

## 1. Executive Summary

Reform fails not because the diagnosis is wrong, but because the delivery is unstructured.

Most reform initiatives launch with energy and ambition, then dissolve into the existing operating rhythm within 60 days. The reason is not resistance to change. It is the absence of a structured delivery system that can survive competing priorities, leadership turnover, and organisational gravity.

This system provides:

```txt
30/60/90 Delivery Architecture — phased reform with clear milestones and success criteria
Reform Governance Structure — who oversees, who decides, who delivers
Progress Measurement System — objective indicators that cannot be gamed
Course Correction Protocol — how to adjust without losing momentum
```

It is designed to make reform structural rather than episodic — not a project with an end date, but a new operating rhythm.

---

## 2. Strategic Context

Reform initiatives fail in five predictable patterns:

```txt
1. Launch without landing — high-energy launch, no delivery mechanism. Momentum fades within 30 days.

2. Scope creep — reform expands beyond original mandate. Resources dilute. Nothing completes.

3. Leadership dependency — reform depends on a single champion. Champion leaves, reform dies.

4. Measurement theatre — progress reported subjectively. No objective indicators. Failure hidden.

5. Reversion — reform achieves initial goals. Without structural embedding, organisation reverts within one cycle.
```

### The Reform Gravity Curve

```txt
Day 1-30:   High energy, clear focus, visible progress
Day 31-60:  Competing priorities emerge. Energy begins to fade.
Day 61-90:  Organisational gravity pulls toward old patterns.
Day 91+:    Without structural embedding, reform is absorbed or abandoned.

The 30/60/90 architecture is designed to counteract this curve.
```

## 3. Core Model — 30/60/90 Delivery Architecture

### 3.1 Phase 1 — Foundation (Days 1-30)

**Objective:** Establish authority, diagnose current state, design target state, launch first interventions.

```txt
Week 1 — Authority and Mandate
  — Reform mandate confirmed by board or senior authority
  — Reform owner designated with clear authority boundaries
  - Reform governance structure installed (steering group, delivery team)
  — Stakeholder map completed

Week 2 — Current State Assessment
  — Complete diagnostic using Institutional Diagnostics Toolkit
  — Identify top 3-5 reform priorities
  — Assess reform capacity and constraints
  — Risk assessment completed

Week 3 — Target State Design
  — Define target state for each priority
  — Design intervention path for each
  — Sequence interventions using Intervention Matrix rules
  — Resource plan completed

Week 4 — Launch
  — Launch first intervention(s)
  — Install progress measurement system
  — Communicate reform plan to all stakeholders
  — First steering group review completed
```

**Phase 1 Deliverables:**

```txt
✓ Reform mandate documented and published
✓ Reform owner and governance structure in place
✓ Current state assessment completed
✓ Top 3-5 priorities identified and sequenced
✓ Target state defined for each priority
✓ First intervention launched
✓ Progress measurement system installed
✓ Stakeholder communication completed
```

**Phase 1 Success Criteria:**

```txt
Reform mandate is unambiguous and published
Top priorities are agreed by all stakeholders
First intervention is active within 30 days
Measurement system is producing objective data
```

---

### 3.2 Phase 2 — Momentum (Days 31-60)

**Objective:** Build momentum through visible progress. Address resistance. Adjust based on early learning.

```txt
Week 5-6 — First Intervention Delivery
  — Complete first intervention(s)
  — Measure and document outcomes
  — Capture lessons learned
  — Communicate progress to stakeholders

Week 7-8 — Course Correction
  — Review progress against plan
  — Adjust interventions based on early results
  — Address resistance points identified
  — Launch second wave of interventions
```

**Phase 2 Deliverables:**

```txt
✓ First intervention(s) completed with documented outcomes
✓ Progress report published
✓ Course corrections documented and implemented
✓ Second wave interventions launched
✓ Resistance points identified and addressed
```

**Phase 2 Success Criteria:**

```txt
First intervention outcomes are measurable and positive
Course corrections are based on evidence, not opinion
Second wave is launched before momentum fades
Stakeholder support is maintained or increased
```

---

### 3.3 Phase 3 — Embedding (Days 61-90)

**Objective:** Embed reforms into operating rhythm. Transition ownership from reform team to business-as-usual. Install monitoring to prevent reversion.

```txt
Week 9-10 — Structural Embedding
  — Update policies, processes, and systems to reflect new state
  — Transfer ownership from reform team to operational owners
  — Train teams on new operating model
  — Install monitoring systems

Week 11-12 — Stabilisation and Handover
  — Complete all remaining interventions
  — Conduct final assessment against target state
  — Document lessons learned
  — Transition reform governance to business-as-usual governance
  — Celebrate and communicate completion
```

**Phase 3 Deliverables:**

```txt
✓ Policies, processes, and systems updated
✓ Ownership transferred to operational owners
✓ Training completed
✓ Monitoring systems active
✓ Final assessment completed
✓ Lessons learned documented
✓ Reform governance transitioned to BAU
```

**Phase 3 Success Criteria:**

```txt
All priority interventions completed
Target state achieved or on clear trajectory
Ownership successfully transferred
Monitoring systems detecting drift
No reversion to pre-reform state within 30 days of handover
```

## 4. Reform Governance Structure

### 4.1 Governance Roles

```txt
Reform Sponsor (Board level)
  — Provides authority and resources
  — Removes obstacles beyond reform owner's authority
  — Reviews progress monthly

Reform Owner (Senior leader)
  — Accountable for delivery
  — Manages reform team
  — Reports to steering group

Reform Team (Dedicated or part-time)
  — Executes interventions
  — Measures progress
  — Identifies risks and issues

Steering Group (Cross-functional)
  — Reviews progress monthly
  — Makes priority and resource decisions
  — Ensures alignment with institutional strategy
```

### 4.2 Decision Rights

```txt
Reform Owner decides:
  — Intervention sequencing within approved plan
  — Resource allocation within approved budget
  — Team composition and task assignment

Steering Group decides:
  — Scope changes beyond original mandate
  — Additional resource requests
  — Priority conflicts between reform and BAU

Reform Sponsor decides:
  — Reform continuation, pause, or termination
  — Escalated issues that steering group cannot resolve
  — Major scope or timeline changes
```

## 5. Progress Measurement System

### 5.1 Measurement Dimensions

```txt
Progress:    % of planned interventions completed (target: on schedule)
Quality:     % of interventions meeting success criteria (target: >80%)
Impact:      % of target state achieved (target: improving each phase)
Sustainability: % of reforms embedded after 90 days (target: >90%)
Health:      Reform team morale, stakeholder support, resistance level
```

### 5.2 Reporting Cadence

```txt
Weekly:     Reform team internal progress check
Bi-weekly:  Progress update to reform owner
Monthly:    Progress report to steering group
Quarterly:  Impact assessment to board
```

### 5.3 RAG Status Definitions

```txt
GREEN:  On schedule. No issues requiring escalation.
AMBER:  Behind schedule or minor issues. Mitigation in place.
RED:    Significantly behind schedule or major issues. Escalation required.
```

## 6. Course Correction Protocol

### 6.1 When to Correct

```txt
Any intervention is behind schedule by >2 weeks
Any intervention is not meeting success criteria
New information changes the understanding of the problem
Stakeholder resistance is blocking progress
Resources are insufficient to maintain momentum
```

### 6.2 Correction Process

```txt
Step 1: Assess — what changed, why, and what is the impact?
Step 2: Option generation — what are the possible responses?
Step 3: Decision — which option, with what timeline, with what resources?
Step 4: Communication — who needs to know about the change?
Step 5: Implementation — execute the correction
Step 6: Monitor — did the correction work?
```

### 6.3 Correction Anti-Patterns

```txt
Doubling down — increasing resources on a failing intervention without changing approach
Abandoning — dropping an intervention at first sign of difficulty without analysis
Scope creep — expanding scope to avoid admitting the original plan was wrong
Silent correction — changing course without communicating to stakeholders
```

## 7. Diagnostic Signals

Signals that phased delivery is needed or failing:

```txt
REFORM_LAUNCH_NO_LANDING  — reform announced but no delivery structure
REFORM_SCOPE_CREEP        — reform expanding beyond original mandate
REFORM_MOMENTUM_LOSS      — reform activity declining after initial launch
REFORM_MEASUREMENT_GAP    — no objective progress indicators
REFORM_REVERSION          — organisation returning to pre-reform patterns
REFORM_FATIGUE            — organisation exhausted by continuous change
```

## 8. Failure Modes

```txt
1. Launch without landing — reform announced with energy, no delivery mechanism exists. Momentum fades within 30 days.

2. Perpetual diagnosis — assessment phase never ends. Reform never moves to intervention.

3. Champion dependency — reform depends on single leader. Leader leaves or loses interest. Reform dies.

4. Scope expansion — reform scope grows beyond delivery capacity. Nothing completes.

5. Measurement substitution — activity measured instead of outcome. Busyness mistaken for progress.

6. Premature handover — reform transferred to BAU before it is structurally embedded. Reversion within one cycle.
```

## 9. Application Playbook

### Step 1 — Establish Reform Mandate

Secure board-level mandate. Document scope, authority, and resources.

**Deliverable:** Reform Mandate Document

### Step 2 — Install Governance

Assign reform sponsor, owner, team, and steering group. Define decision rights.

**Deliverable:** Reform Governance Charter

### Step 3 — Design 30/60/90 Plan

Define specific interventions, milestones, and success criteria for each phase.

**Deliverable:** 30/60/90 Reform Plan

### Step 4 — Launch Phase 1

Execute foundation phase. Complete current state assessment and launch first interventions.

**Deliverable:** Phase 1 Completion Report

### Step 5 — Review and Adjust

At day 30, conduct phase review. Adjust plan based on learning. Launch Phase 2.

**Deliverable:** Phase Review Report + Adjusted Plan

### Step 6 — Complete and Embed

Complete all interventions. Transfer ownership. Install monitoring.

**Deliverable:** Reform Completion Report + Monitoring Dashboard

```txt
Week 1: Mandate confirmed. Governance installed.

Week 3: Target state designed. 4 priorities identified.

```

```txt
Decision rights matrix completed and published.
Process redesign launched for top 3 friction points.
Resistance from one department head addressed through direct engagement.
```

### Phase 3 (Days 61-90)

```txt
All 4 priority interventions completed.
Policies updated. Training delivered.
Ownership transferred to operational leaders.
Monitoring dashboard activated.
```

### Outcome

```txt
Decision latency reduced by 35%
Execution friction reduced by 28%
Culture alignment score improved by 22%
All reforms embedded and monitoring active at day 90
No reversion at 120-day review
```

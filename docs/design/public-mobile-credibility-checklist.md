# Public mobile credibility checklist

Use this before public outreach or any mobile release review.

1. Fixed headers must not collide with the first section.
2. Two-column technical rows must stack on mobile.
3. Body text minimum: 14px; preferred 15–16px.
4. Mono labels may be small, but never carry the primary explanatory text.
5. No public page should require horizontal reading compression.
6. Every public strategic surface must be reachable from the homepage, header, or footer unless it is intentionally private.
7. Gated or qualified pages must say so plainly.
8. Do not make WORM, blockchain, or external verification claims unless those capabilities are live.

# LinkedIn Company Page Publishing — Integration Guide

## Overview

This document describes the official LinkedIn API integration for publishing posts to the Abraham of London LinkedIn company page.

**Status:** Scaffold only — not activated. Requires LinkedIn developer approval and credential configuration before use.

## Prerequisites

### LinkedIn Developer Account

1. Go to https://developer.linkedin.com/
2. Create or sign in to your LinkedIn Developer account
3. Create a new app in the Developer Portal
4. Request the **Marketing Developer Platform** product (required for `w_organization_social`)

### Required Permissions

| Permission | Purpose |
|---|---|
| `w_organization_social` | Create, read, and update posts as the organization |
| `r_organization_social` | Read organization posts and engagement data |

### Organisation Author URN

LinkedIn identifies organisations by URN in the format:

```
urn:li:organization:115850136
```

To find your organisation ID:
1. Go to your LinkedIn company page
2. The ID is in the page URL: `https://www.linkedin.com/company/{ID}/`
3. For Abraham of London: `urn:li:organization:115850136`

### Authenticated User Requirements

- The authenticated user must be a **Super Admin** or **Content Admin** of the LinkedIn company page
- The access token must be generated by a user with these roles
- Token must have the `w_organization_social` scope

## Environment Variables

```bash
# Enable LinkedIn API publishing (must be "true" to activate)
LINKEDIN_PUBLISHING_ENABLED=false

# LinkedIn OAuth credentials
LINKEDIN_CLIENT_ID=your_client_id
LINKEDIN_CLIENT_SECRET=your_client_secret

# Organisation identifier
LINKEDIN_ORGANIZATION_ID=115850136

# Access token (or OAuth token storage path)
LINKEDIN_ACCESS_TOKEN=your_access_token
```

## Supported Post Types (v1)

### Text-Only Post

```json
{
  "author": "urn:li:organization:115850136",
  "commentary": "Post body text here...",
  "visibility": "PUBLIC",
  "distribution": {
    "feedDistribution": "MAIN_FEED",
    "targetEntities": [],
    "thirdPartyDistributionChannels": []
  },
  "lifecycleState": "PUBLISHED",
  "isReshareDisabledByAuthor": false
}
```

### Link Share Post

```json
{
  "author": "urn:li:organization:115850136",
  "commentary": "Post body text here...",
  "visibility": "PUBLIC",
  "distribution": {
    "feedDistribution": "MAIN_FEED",
    "targetEntities": [],
    "thirdPartyDistributionChannels": []
  },
  "content": {
    "article": {
      "source": "https://www.abrahamoflondon.org/...",
      "title": "Link Title",
      "description": "Link description..."
    }
  },
  "lifecycleState": "PUBLISHED",
  "isReshareDisabledByAuthor": false
}
```

## Not Supported in v1

- **Images** — Requires `w_organization_social` with image upload (separate endpoint)
- **Video** — Requires video upload endpoint and additional permissions
- **Carousels** — Not available via Posts API
- **Documents** — Not available via Posts API
- **Scheduled posting** — Requires separate scheduling infrastructure

## API Endpoints

### Publish a Post

```
POST /api/admin/outbound/linkedin/publish
```

**Request:**
```json
{
  "filename": "06-execution-problem-vs-authority-problem.mdx"
}
```

**Response (success):**
```json
{
  "ok": true,
  "linkedinPostUrl": "https://www.linkedin.com/company/115850136/posts/...",
  "message": "Post published successfully."
}
```

**Response (not configured):**
```json
{
  "ok": false,
  "error": "LINKEDIN_PUBLISHING_NOT_CONFIGURED",
  "message": "LinkedIn API publishing is not configured..."
}
```

## Token Refresh & Storage

Before activating the API, design and implement:

1. **Token storage** — Store tokens securely (environment variables or encrypted storage)
2. **Token refresh** — LinkedIn access tokens expire after 60 days. Implement a refresh mechanism using the refresh token
3. **Token rotation** — Rotate tokens before expiry to avoid service interruption

## Security Considerations

- Never expose access tokens in client-side code
- All API calls must be server-side only
- Audit log every publish action
- Require admin authentication for all publishing endpoints
- Validate post content before sending to LinkedIn
- Never publish accidentally — require explicit confirmation

## Testing

Before going live:

1. Verify organisation ID is correct
2. Test with a single post in a development environment
3. Verify the post appears on the company page
4. Test token refresh flow
5. Test error handling (invalid token, expired token, network failure)

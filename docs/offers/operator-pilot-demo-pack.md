# Operator Pilot Demo Pack

Status: LIVE_SURFACES_ONLY  
Purpose: prove this is not a diagnostic toy

## Demo Rule

Do not show the whole product.

This track exists to prove one claim:

`A serious decision can enter the system, be tested under pressure, return a governed finding, and continue into accountable execution.`

## Demo Sequence

### 1. Homepage refusal thesis

Show:

- refusal-first positioning
- evidence and authority language
- the promise that the system can refuse to proceed

Say:

`This is not a dashboard and not a generic advisory front end. The system begins by testing whether a real decision is ready to move.`

### 2. Fast Diagnostic live

Show:

- one real decision under pressure
- default path first
- optional evidence only at the end

Say:

`The entry point stays fast. Extra evidence is available, but it does not burden the default path.`

### 3. Fast Result

Show:

- contradiction
- required move
- checkpoint
- governed memory / return-brief continuity

Say:

`This is the first important conversion. The output is not a motivational summary. It is a governed finding with a next move and a future accountability point.`

### 4. Decision Centre

Show:

- case continuity
- checkpoint state
- return brief availability
- boardroom / retainer readiness cues

Say:

`The result does not disappear. It becomes an operating case with memory, follow-up, and escalation paths.`

### 5. Executive Reporting result

Show:

- evidence posture
- decision framing
- next-step bridge

Say:

`This is where the product stops looking like a consumer diagnostic and starts looking like decision infrastructure.`

### 6. Strategy Room

Show:

- evidence carried from ER
- checkpoint governance
- counsel posture
- execution reality

Say:

`This is not a brainstorm room. It is a decision chamber built around trade-off, ownership, and consequence.`

### 7. Return Brief

Show:

- reopened condition
- continuity from prior record
- what changed / what failed to change

Say:

`The system does not treat inaction as a reset. It confronts the gap between commitment and reality.`

### 8. Evidence Standards close

Show:

- human review gate
- what is publishable
- what is never claimed

Say:

`The close is not hype. The close is standards. This is how the system earns the right to make strong claims later.`

## Core Narrative

Use this line:

`Bring one decision your organisation cannot afford to keep circling. The system will test it, identify the contradiction, issue a required move, schedule accountability, and show what changes.`

## What This Demo Must Prove

- the product can refuse weak or incomplete movement
- the result becomes a governed case, not a disposable output
- follow-up is part of the institution, not an operator memory trick
- the system has evidence discipline
- the system has execution continuity

## What Not To Show In This Track

- full retained oversight command
- every admin surface
- boardroom archive depth unless directly needed
- speculative portfolio intelligence language
- any £50k positioning

## Best Close

`This is the operator pilot. We are not asking you to buy a platform tour. We are asking you to bring one real decision and let the system prove whether it can govern movement better than your current loop.`

# Controlled £50k Demo Route Map

Classification: `CONTROLLED_50K_CONVERSATION_READY`

This is not a sales script. This is a documented route through the institutional corridor for a controlled high-value engagement conversation.

## Route

Homepage → Fast Diagnostic → Executive Reporting → Strategy Room → Boardroom → Oversight Command → Portfolio Memory → Counsel Status → Proof Pack / PDF

---

### Step 1: Homepage (`/`)

**Value visible:** Product positioning, trust surface, diagnostic entry point.
**Evidence used:** None — public surface.
**Suppressed:** Nothing.
**Must not claim:** "Enterprise-ready £50k", "always-on governance", "automated oversight."
**If data is thin:** N/A — this is a static surface.

---

### Step 2: Fast Diagnostic (`/diagnostics/fast`)

**Value visible:** Condition classification (authority/definition/execution/instability), signal strength, trajectory, consequence estimate, earned product progression.
**Evidence used:** User-reported inputs only. No external data.
**Suppressed:** Internal scoring weights, threshold logic, formula mechanics.
**Must not claim:** "Verified outcome", "guaranteed result", "independently verified."
**If data is thin:** Show "Insufficient signal" with honest caveat. Do not fabricate condition.

---

### Step 3: Executive Reporting (`/diagnostics/executive-reporting/run`)

**Value visible:** Canonical assessment with sections, irreversibility estimate, cost-of-inaction estimate, financial exposure band, stakeholder pressure, advantage terrain, intervention suggestions, boardroom qualification.
**Evidence used:** Diagnostic intake + constitutional assessment. System-inferred.
**Suppressed:** Raw canonical payload (filtered through `toExecutiveReportingPublicResult`), kernel internals, graph structure.
**Must not claim:** "Verified improvement", "continuous monitoring." Call estimates "estimates."
**If data is thin:** Executive report still renders with available data. Thin sections show "Insufficient evidence for this section."

---

### Step 4: Strategy Room (`/strategy-room` → `/strategy-room/session/[id]`)

**Value visible:** Execution state, decision frame, constraint map, intervention stack, stakeholder pressure (via composer), scenario pressure, what-changed tracking, checkpoint governance, irreversibility estimate, counsel status panel.
**Evidence used:** Canonical snapshot + spine + stakeholder map + simulation. Server-side only.
**Suppressed:** Simulation formula, probability model, threshold logic, kernel internals. Stakeholder identities only shown when user-provided.
**Must not claim:** "Automated oversight", "fully autonomous." Scenario estimates must include "not independently verified" caveat.
**If data is thin:** Stakeholder pressure block hidden. Suppression notice shown. Scenario pressure marked thin.

---

### Step 5: Boardroom (`/boardroom`)

**Value visible:** Archive statistics, institutional case qualification state, stakeholder exposure, scenario pressure, decision record posture, contradiction pressure band, boardroom readiness.
**Evidence used:** Institutional case resolver + canonical composer (stakeholder, simulation, contradiction metrics).
**Suppressed:** Raw graph data, kernel output, pricing/entitlement logic, operator notes.
**Must not claim:** "Board-ready" unless qualification state supports it. "Verified" only for verified outcomes.
**If data is thin:** "No boardroom archive is available for this scope yet." Stakeholder/scenario sections hidden.

---

### Step 6: Oversight Command (`/oversight`)

**Value visible:** Constitutional health posture, institutional case metrics, sponsor-safe command summary, cadence posture, cadence history, retained memory loss panel.
**Evidence used:** Constitutional command centre data (public-safe posture only), institutional case service, cadence service.
**Suppressed:** Raw constitutional scores, threshold bands, internal dimension names. Health strip shows only band + posture.
**Must not claim:** "Continuous monitoring", "always-on governance." Cadence is "scheduled" not "automated."
**If data is thin:** Health strip hidden. "Retained oversight command is not yet established."

---

### Step 7: Oversight Brief (`/oversight/brief/[cycleId]`)

**Value visible:** Executive summary, cost-of-inaction, pattern recurrence, irreversibility, strategic options, cycle projection, contradiction pressure, stakeholder friction, oversight signals, structured actions, suppression summary.
**Evidence used:** Oversight brief composer + canonical intelligence composer. All source-labelled.
**Suppressed:** Raw respondent text, operator notes, graph mechanics, formula logic.
**Must not claim:** "Verified improvement" without verification record. All estimates must be caveated.
**If data is thin:** Brief renders with available sections only. Missing sections omitted entirely, not faked.

---

### Step 8: Portfolio Memory (`/oversight/portfolio`)

**Value visible:** Cross-case aggregates, recurring patterns, contradiction classes, decision velocity, checkpoint posture, counsel escalations, boardroom dossiers, cadence reliability, cross-scope patterns, portfolio scenario pressure, stakeholder recurrence.
**Evidence used:** Portfolio memory surface + cross-org pattern intelligence + role-dynamic patterns.
**Suppressed:** Raw respondent data, small-sample details, individual case text. Suppression notice always visible.
**Must not claim:** Sector intelligence without minimum sample. Role patterns without "system-inferred" label.
**If data is thin:** "Portfolio intelligence is based on limited observations. Trend claims require additional cycles."

---

### Step 9: Counsel Status (`/counsel/status`)

**Value visible:** Counsel case timeline, case status, stakeholder pressure summary, institutional case corridor state.
**Evidence used:** Counsel case service + canonical composer (stakeholder pressure).
**Suppressed:** Raw respondent statements, internal reviewer notes, counsel notes.
**Must not claim:** "Counsel is active" unless case exists. "Counsel recommended" only when triggers are met.
**If data is thin:** Stakeholder pressure section hidden. "Counsel is not yet warranted by the evidence available."

---

### Step 10: Proof Pack / PDF (`/account/proof-pack`)

**Value visible:** Retained outcome summary, evidence posture, forward action, suppression summary.
**Evidence used:** Retained outcome summary service + suppression ledger.
**Suppressed:** Raw case data, operator notes, internal audit trail.
**Must not claim:** "This is a substitute for verification, counsel, or oversight." Disclaimer required.
**If data is thin:** "Insufficient retained history for proof generation."

---

## What to say if the buyer asks "Is this £50k ready?"

"Yes, for a controlled qualified scope. The infrastructure is live: the memory layer is role-safe, the cadence path is scheduler-compatible, the suppression ledger is auditable, and the institutional intelligence corridor is wired through a single canonical composer. Generalised £50k claims remain blocked until live retained history matures — and we will not make that claim until the evidence supports it."

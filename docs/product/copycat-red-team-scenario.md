# Copycat Red-Team Scenario

**Date:** 9 May 2026
**Purpose:** Identify what a competitor would copy first and what they cannot copy.

---

## Scenario

A well-funded competitor sees Abraham of London's product and decides to build a competing "Decision Infrastructure" platform. They have:
- $5M in funding
- 6 engineers
- 3 months to MVP
- Access to the same LLM APIs
- No existing user base
- No accumulated data

---

## What They Copy First (Days 1-30)

### 1. The Language
They copy the category language: "Decision Infrastructure", "Governed Synthesis", "Evidence Carried Forward", "Decision Memory", "Cost of Inaction", "Contradiction Detected", "Checkpoint Overdue".

**Impact:** Category confusion. They look like a credible alternative on the surface.
**Defence:** Category language is not a moat. The moat is what the language describes.

### 2. The Result-Page Structure
They copy the Fast Diagnostic result page: Finding → Pattern → Required Move → Cost of Inaction → Commitment → Next Route → Evidence & Governance.

**Impact:** Their result page looks similar to yours.
**Defence:** The structure is visible. What fills it (your synthesis engine, arbiter tournament, contradiction graph) is not.

### 3. The 3-Question Diagnostic Format
They copy the Fast Diagnostic format: "What decision?", "Who owns it?", "What's the consequence?"

**Impact:** They have a front-line conversion surface.
**Defence:** Without the checkpoint system, evidence spine, and contradiction graph, their diagnostic is a one-off. Yours feeds an entire ladder.

### 4. The "Return Brief" Concept
They copy the idea of confronting users with unresolved decisions.

**Impact:** They have a retention mechanism.
**Defence:** Without the checkpoint system, trigger evaluation, and evidence carry-forward, their return brief is a reminder email. Yours is a governed confrontation.

### 5. The "Decision Centre" Concept
They copy the multi-case console idea.

**Impact:** They have a dashboard.
**Defence:** Without the evidence spine, checkpoint integration, and governed memory, their console shows empty cases. Yours shows accumulated evidence.

### 6. The "Checkpoint" Concept
They copy the idea of scheduling follow-ups.

**Impact:** They have a basic reminder system.
**Defence:** Without the 7-surface integration, response classification, and overdue detection, their checkpoints are calendar reminders. Yours are governed enforcement.

---

## What They Cannot Copy (Days 31-90)

### 1. The Evidence Memory Chain
80 fields across 8 stages with source-labelled rendering. This is architectural — it took years to design and implement.

**Their alternative:** They can build a basic persistence layer in 30 days. They cannot build the field-by-field closure, the cross-surface inheritance, or the governed memory rendering in 90 days.

### 2. The Checkpoint Response History
Every checkpoint response (COMPLETED, BLOCKED, ABANDONED, DISPUTED) is accumulated per user. This data gets more valuable with time.

**Their alternative:** They can build a checkpoint system in 30 days. They have zero response history. Every user starts from zero.

### 3. The Contradiction Graph
Temporal, cross-assessment, decay-aware. Every evaluation extends the graph. Unresolved contradictions compound in severity.

**Their alternative:** They can build a graph data structure in 30 days. They have zero nodes and zero edges. Every user starts with an empty graph.

### 4. Cross-Assessment Intelligence
The kernel detects when signals from different assessments contradict each other. Purpose says "strong identity" but Constitutional says "authority is unclear."

**Their alternative:** They can build the detection logic in 30 days. They have zero users with multiple completed assessments. The intelligence requires data that takes months to accumulate.

### 5. Decision Velocity Data
Time from diagnosis → commitment → action → outcome. Requires checkpoint response data over time.

**Their alternative:** They can build the tracking in 30 days. They have zero velocity data. Every user starts from zero.

### 6. The Strategy Room Enforcement State
Directive system, consequence tracking, avoidance pattern detection, escalation triggers, checkpoint integration. Requires the full ladder.

**Their alternative:** They can build a session concept in 30 days. They have no enforcement state, no escalation triggers, no checkpoint integration. It's a chat room.

### 7. The Counsel Escalation Workflow
Evidence package prefill, structured intake, operator review, case lifecycle. Requires the evidence spine and operator adoption.

**Their alternative:** They can build an intake form in 7 days. They have no evidence package to prefill, no operator workflow, no case lifecycle.

---

## Month 3-6: The Gap Widens

By month 3, your system has:
- More checkpoint responses per user
- More contradiction graph nodes per user
- More cross-assessment detections per user
- More decision velocity data points per user
- More counsel cases per user
- More oversight cycles per user

Their system has:
- The same UI structure
- The same language
- Zero accumulated data
- Zero workflow dependency
- Zero user trust

**The gap is not in the code. The gap is in the data.**

---

## Month 6-12: The Moat Hardens

By month 6, your system has:
- Verified outcome data (if outcome verification loop is built)
- Boardroom archive
- Pattern recurrence history
- Organisation divergence history
- Portfolio pattern detection
- Anonymised benchmarks

Their system has:
- A UI that looks similar
- Zero outcome data
- Zero pattern history
- Zero benchmarks

**At this point, competing is irrational.** They would need to spend 12-24 months accumulating data that you already have. Acquisition or partnership becomes the rational move.

---

## What This Means

| Timeline | Their Position | Your Position |
|----------|---------------|---------------|
| Month 1 | Copy language + UI structure | Existing architecture |
| Month 3 | Basic functional clone | Accumulated data per user |
| Month 6 | Stalled — no data moat | Verified outcomes, benchmarks |
| Month 12 | Acquisition target | Category leader |

**The defence is not secrecy. The defence is accumulation.** Build features that generate data per user over time. Every day a user stays, their data makes your system more valuable and theirs harder to leave.

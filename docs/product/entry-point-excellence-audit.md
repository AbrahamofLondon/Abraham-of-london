## Remaining weakest entry points

1. **Memory invitation across mid-tier assessments** — Constitutional, Team, and Enterprise assessments inherit context but don't explicitly promise tracking. Add "This decision will be tracked" language to result surfaces.
2. **Consequence on entry pages** — Most entry pages describe what they test but don't show consequence until results. Consider adding a micro-consequence signal.
3. **Hub memory signal** — The diagnostics hub has no return-user detection or memory preview. A returning user should see acknowledgement of prior sessions.

## Expectation-Exceeding Pass (2026-05-07)

### What changed

1. **Restricted SR enrollment email:** RESTRICTED users now receive a different email ("case recorded, admission restricted") with missing evidence, repair actions, return path, and case reference. ADMITTED users still receive "access confirmed." No access-confirmed language is sent to restricted users.

2. **AdmissionNotice component** (`components/product/AdmissionNotice.tsx`): Reusable institutional notice supporting ADMITTED/RESTRICTED/BLOCKED/REPAIR_REQUIRED/REVIEW_REQUIRED. Shows evidence tier, authority status, continuity, case reference, repair actions, return path. Case preservation message: "Your case has been preserved. You do not need to start again."

3. **Return Brief continuity header:** Now shows "You are not starting again. The system remembers this case." with session reference, generation date, and trajectory status. Signal continuity section classifies signals as worsening/repeated/improving/new based on trajectory state and contradiction presence.

4. **Proof classification UX:** Evidence pages show "Static proof asset" label. Homepage proof blocks show `data-proof-status` and visible "Demonstration patterns" label when in fallback mode. AccuracyMetricsBlock shows `data-sample-size` for transparency.

5. **Strategy Room entry upgraded from WEAK to PARTIAL:** Server-side admission enforcement now blocks execution without evidence. AdmissionNotice provides structured restriction experience. Admission metadata attached to all session records.

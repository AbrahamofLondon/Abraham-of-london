# Homepage Message Architecture Audit

Date: 2026-05-09

## Verdict

Current message architecture is not market-ready. The page reaches for category authority before it has earned narrative clarity, and it tries to close before it has delivered proof.

## Current Sequence

1. Category claim: `Decision Infrastructure by Abraham of London`
2. Refusal capability
3. Refusal demo
4. Competitor/category contrast
5. Product ladder
6. Governance capability list
7. Audience pathways
8. Trust doctrine
9. Final CTA
10. Applied case evidence
11. Flagship output
12. Fit / seriousness
13. Miscellaneous route strips

## Strategic Failures

- The page tells the user the institution can refuse, but later behaves like a route menu.
- The page claims earned progression, then offers direct access into later-stage routes.
- The page closes once inside `CategoryFrontDoor`, then resumes explaining itself afterward.
- The page talks about memory and continuity more than it shows them.

## Recommended Sequence

1. Category declaration
2. Why this matters now
3. Refusal demonstration
4. What the user gets back
5. How evidence becomes memory
6. Start with evidence
7. Flagship outputs and escalation
8. Trust architecture
9. Qualified audiences
10. Final CTA

## Headline Direction by Block

1. Hero: category plus refusal
   `The decision system that can refuse to proceed.`
2. Why it matters:
   `The real cost is not indecision. It is ungoverned action wearing the mask of progress.`
3. Demo:
   `Watch a serious decision get restricted.`
4. Output artifact:
   `What the system returns when the case is real.`
5. Memory:
   `What enters the record does not reset.`
6. Start path:
   `Bring one decision under pressure.`
7. Flagships:
   `When the evidence justifies consequence and intervention.`
8. Trust:
   `How this system refuses, labels, suppresses, and challenges.`
9. Qualification:
   `For operators carrying consequence, not browsers collecting opinions.`
10. Final CTA:
   `Test one decision.`

## Required Removals

- Remove the first final CTA block from the front-door sequence.
- Remove or heavily compress the buyer-path segmentation block.
- Remove one of the two late trust/link strips.

## Bottom Line

The page should feel like a threshold into governed decision handling. Right now it feels like a threshold, a doctrine page, a product ladder, and a route menu all at once.

# Homepage Mobile Post-Rebuild Checklist

Date: 2026-05-09
Status: pending visual QA

## Mobile Checks

- Verify hero headline fits cleanly without awkward orphan lines.
- Verify hero support copy remains readable and does not collapse into a doctrinal wall.
- Verify the `Test a Decision` CTA is visible in the first viewport on common phone heights.
- Verify `See how refusal works` remains tappable and visually subordinate.
- Verify the market-defect cards stack without feeling repetitive.
- Verify the refusal demo keeps clear section rhythm and label readability.
- Verify the output artifact remains legible in one column.
- Verify the memory chain reads as progression rather than a long list.
- Verify the earned-progression states remain understandable when stacked vertically.
- Verify trust cards do not exceed acceptable scroll fatigue.
- Verify the compact homepage footer does not reintroduce route clutter.
- Verify no horizontal overflow exists.
- Verify the fixed header does not overpower the first viewport.
- Verify anchor navigation to `/#hero` and `/#refusal-demo` behaves correctly.

## Acceptance

Mobile is acceptable only if the homepage still feels like a controlled institutional threshold rather than a compressed manifesto.

# Homepage Post-Rebuild Message Map

Date: 2026-05-09

## Message Sequence

1. **Hero**
   - What is this?
   - `The decision system that can refuse to proceed.`
   - Action: `Test a Decision`

2. **Market Defect**
   - Why do ordinary tools fail?
   - They help too early, after weak framing and weak authority have already been accepted.

3. **Refusal Demo**
   - What does this system do differently?
   - It can stop escalation, state what is missing, and retain the correction path.

4. **Output Preview**
   - What comes back when I submit a decision?
   - Finding, contradiction, required move, consequence warning, checkpoint, memory note.

5. **Decision Memory**
   - What does the system remember?
   - The case record carries forward what was said, what was committed, and whether the user acted.

6. **Earned Progression**
   - Why can I not select later-stage products directly?
   - Because the next layer is earned by evidence, not chosen from a menu.

7. **Trust Architecture**
   - Why can it be trusted?
   - Source labels, estimate labels, refusal discipline, no fabricated verification, suppression of unsafe material.

8. **Final CTA**
   - What should I do first?
   - `Test a Decision`

## Message Discipline Rules

- One dominant claim: refusal.
- One dominant action: submit evidence through `Test a Decision`.
- No route-menu framing in the homepage body.
- No late-stage product as a normal starting path.
- No trust slogan without a rule or visible artifact nearby.

# Homepage Rebuild Recommendation

Date: 2026-05-09

## Goal

Rebuild the homepage as the category front door for Decision Infrastructure. It should feel institutional, governed, and earned. It should route users into evidence submission, not product shopping.

## Recommended Section Order

1. Hero — Category + refusal capability
2. Refusal Engine Demo — visible governed review
3. What the System Returns — concrete output artifact
4. What the System Remembers — decision memory / continuity proof
5. Start with Evidence — Fast Diagnostic entry
6. Flagship Outputs — Executive Reporting and Strategy Room
7. Trust Architecture — labels, suppression, challenge, no fabricated verification
8. Serious Use Cases — operators, founders, executives, institutions
9. Final CTA — test one decision

## Section Jobs

| Section | Job | Headline Direction | CTA Direction | Visual Direction | Trust / Evidence Function | Mobile Treatment |
| --- | --- | --- | --- | --- | --- | --- |
| Hero | Declare category and first action | `The decision system that can refuse to proceed.` | Primary: `Test a decision` | Calm, large, fewer micro-lines | Establish seriousness without overclaim | Reduce support copy |
| Refusal demo | Prove this is governed, not generative theatre | `Watch a decision get restricted.` | Secondary: `See what the system checks` | Keep current demo, sharpen labels | Strong refusal proof | Keep stacked cards |

## What to Remove

- Front-door buyer pathways block in current form
- Front-door premature final CTA
- Duplicate late trust strip
- Full exposed ladder from first encounter

## What to Add

- One output artifact
- One memory / continuity artifact
- One explicit line on what happens after submission

## Visual Direction

- Preserve the dark institutional palette and serif/mono contrast.
- Reduce the number of bordered text grids.
- Replace at least two abstract text sections with product proof.
- Use gold only for authority and action, not as repeated accent noise.

## Copy Direction

- Move from category poetry to governed plain language.
- Keep the refusal line.
- Replace some "what we are not" language with "what the user gets and why it is different."
- Shorten trust doctrine by 30 to 40 percent.

## Bottom Line

The rebuild should not become more decorative. It should become stricter. Fewer sections, fewer claims, harder proof, cleaner routing.

## Implemented Structure

The current rebuild pass implements this eight-section order:

1. Hero
2. Market Defect
3. Refusal Demo
4. Output Preview
5. Decision Memory
6. Earned Progression
7. Trust Architecture
8. Final CTA

Key implementation decisions:

- `Test a Decision` remains the only dominant CTA in the homepage body.
- Later-stage products are described as conditional or earned, not presented as casual starting paths.
- The homepage now shows both a sample return artifact and a continuity chain so refusal and memory are visible instead of only asserted.

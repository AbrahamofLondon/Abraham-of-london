# IP-Safe Surfacing Map

**Date:** 9 May 2026
**Purpose:** Define what to surface publicly, what to surface privately, and what to suppress for every major advantage.

---

## Surfacing Strategy

| Advantage | Public Surface | Private/Operator Surface | Suppress |
|-----------|---------------|------------------------|----------|
| Contradiction Graph | User-safe contradiction map showing active conflicts with severity bands and plain-English explanations | Full graph with node/edge detail, dependency chains, health metrics for operator | Raw graph mechanics, node/edge data structures, decay algorithms |
| Arbiter Tournament | "Quality checks passed" badge showing pass/fail count per analysis | Full arbiter trace showing which rules passed/failed and why | Rule list, rule weights, rejection thresholds |
| Decision Kernel Accuracy | Accuracy summary after sufficient data: "System predictions have been X% accurate across Y decisions" | Full bias correction history, per-domain accuracy breakdown | Internal correction logic, prediction/outcome comparison methodology |
| Decision Velocity | User score + trend: "Your average time from diagnosis to action is X days" | Cohort/portfolio analytics, per-surface breakdown | Formula details, scoring methodology |
| Irreversibility Index | Band label + plain-English warning: "This decision is becoming harder to reverse" | Full factor breakdown, trend data | Factor weights, calculation methodology |
| Cross-Assessment Intelligence | Safe explanation: "Your Purpose Alignment and Constitutional Diagnostic reveal a pattern worth examining" | Full signal trace showing which assessments contradict and why | Internal inference chain, contradiction detection logic |
| Pattern Recurrence | "This pattern has appeared before" with count and date of first detection | Full recurrence map with matched cases | Matching algorithm, text comparison methodology |
| Boardroom Dossier | Downloadable PDF dossier with sections, objections, decision paths | Archive of all generated dossiers with status tracking | Dossier generation logic, qualification thresholds |
| Checkpoint System | "Checkpoint scheduled" / "Checkpoint overdue" with response options | Full checkpoint history with response details, classification audit | Response classification mapping, overdue detection logic |
| Evidence Spine | "Evidence carried forward" with source labels and dates | Full evidence trace per case | Internal field mapping, persistence architecture |
| Decision Kernel | "Decision evaluated" with condition and required move | Full kernel output with graph health metrics | Internal kernel architecture, constraint evaluation logic |
| Strategy Room Enforcement | "Execution state: active" with directive and consequence | Full enforcement state with escalation triggers, avoidance patterns | Enforcement heuristics, trigger thresholds |
| Counsel Escalation | "Counsel Review may be warranted" with evidence package summary | Full counsel case with operator notes, response history | Escalation trigger weights, operator review guidelines |
| Outcome Verification | "Outcome recorded" with classification | Full outcome history with evidence trail | Verification methodology, confidence calculation |
| Proof Pack | Downloadable evidence summary for user | Full proof pack with all available data | Generation logic, data selection criteria |

## Language Rules

### Use these phrases
- "Evidence carried forward"
- "Decision memory"
- "Contradiction detected"
- "Checkpoint scheduled"
- "Checkpoint overdue"
- "Route not yet earned"
- "Action not yet verified"
- "Pattern is recurring"
- "Options may be narrowing"
- "Cost of inaction estimated"
- "Based on your stated inputs"
- "Not independently verified"
- "Source: [surface name]"
- "Captured on [date]"
- "User-reported"
- "System-inferred"
- "Previously reported"

### Never use these phrases publicly
- "Our proprietary algorithm"
- "Our kernel computes"
- "Our graph detects"
- "AI determined"
- "The system proved"
- "Verified by our engine"
- "Patent-pending"
- "Machine learning model"
- "Neural network"
- "Deep learning"

### Use these instead
- "The system identified"
- "Based on your evidence"
- "The pattern suggests"
- "Earlier signals indicate"
- "The record shows"
- "What changed since last time"

# Public Communication Disclosure Standard

**Date:** 2026-05-08
**Rule:** Public pages expose decision consequences, not engine internals.

---

## Disclosure Tiers

### Tier 1 — Public Marketing (homepage, public pages, social, press)

**Allowed:**
- Evidence quality, authority clarity, consequence exposure, execution readiness
- Governed review, decision memory, verified follow-up, outcome tracking
- Contradiction detection, evidence grading, admission/refusal
- Cost-of-inaction, pattern recurrence, boardroom readiness
- "The system can refuse to proceed"

**Forbidden:**
- C3 fidelity engine / C3 score
- Contradiction kernel / decision kernel
- Arbiter tournament / arbiter validation
- Action simulation engine
- Synthesis engine / governed synthesis
- Signal continuity algorithm
- Pattern recurrence algorithm details
- Anchor narrative engine
- Micro-tension validation
- Constitutional orchestration engine
- Any named internal function or module

### Tier 2 — Authenticated Product (diagnostic results, Decision Centre, Strategy Room, Return Brief)

**Allowed (additional to Tier 1):**
- Evidence tier labels (single_source, multi_source, outcome_verified)
- Admission status with reasons
- Cognitive state labels
- Signal continuity labels (NEW, REPEATED, WORSENING, etc.)
- Structured action details with evidence basis
- Cost clock with specific amounts
- Decision credit score and trend

**Forbidden:**
- Internal module names
- Scoring algorithm thresholds
- Arbiter constraint specifications
- Raw engine configuration

### Tier 3 — Internal / Admin / Investor

**Allowed (additional to Tier 2):**
- All engine names and internal module references
- Scoring thresholds and algorithm details
- Architecture documentation
- Prisma model references
- API route internals

---

## Rationale

The public impression should be:

> "This institution knows how to test decisions."

Not:

> "This startup has built several clever engines."

The first sells authority. The second invites copying.

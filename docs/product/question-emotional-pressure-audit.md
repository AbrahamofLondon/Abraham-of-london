# Emotional Pressure Audit

> Audit date: 2026-05-08
> Method: Each question classified by emotional effect. Questions that are technically valid but emotionally poor are flagged.

---

## Emotional Movement Framework

The question layer must produce this emotional trajectory:

1. **Recognition** — "this system understands what I am facing"
2. **Discomfort** — "I cannot hide behind vague language"
3. **Precision** — "I must name the real issue"
4. **Accountability** — "I cannot pretend this has no consequence"
5. **Relief** — "the system is helping me face this"
6. **Commitment** — "I know what must happen next"
7. **Continuity** — "the system will remember this"

---

## Emotional Effect Classification Key

| Code | Meaning |
|------|---------|
| CLARIFIES | Helps the user see their situation more clearly |
| DISARMS | Makes confession safe before demanding truth |
| CONFRONTS | Forces the user to face what they are avoiding |
| EXPOSES_CONTRADICTION | Makes two things that cannot both be true visible |
| BUILDS_TRUST | Makes the user believe the system understands them |
| CREATES_PRESSURE | Makes delay or evasion feel costly |
| CREATES_CONFUSION | Wording is unclear or ambiguous |
| FEELS_GENERIC | Could appear in any product |
| FEELS_INTRUSIVE | Asks for more than the relationship warrants |
| FEELS_LOW_VALUE | The answer does not seem to matter |
| FEELS_PREMIUM | The user senses sophistication and intentionality |
| FEELS_BUREAUCRATIC | The user senses form-filling, not diagnosis |

**Emotional trajectory: DISARMS → CONFRONTS → CREATES_PRESSURE → CREATES_PRESSURE**
**Assessment: Excellent.** Pressure escalates correctly. The user starts with permission to confess, ends with accountability. The commitment gate before results is the emotional peak.

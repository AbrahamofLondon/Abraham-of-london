## Part 3 — IP Exposure Assessment

### Safe to disclose publicly
- The system tracks whether decisions move
- The system remembers prior evidence
- The system distinguishes reported, inferred, and verified evidence
- The system can refuse escalation when evidence is insufficient
- The system shows what changed over time
- The system identifies unresolved contradictions
- The system schedules checkpoint review
- The system carries evidence forward between stages
- The system detects when patterns repeat
- The system prices the cost of inaction

### Unsafe to disclose publicly
- Exact scoring formulas and signal weighting
- Prompt structures and LLM integration details
- Contradiction graph node/edge mechanics
- Arbiter tournament rule list
- Kernel self-correction logic and bias computation
- Thresholds for escalation and route decisions
- Internal route arbitration heuristics
- Counsel escalation trigger weights
- Benchmark construction methodology before data maturity
- Irreversibility index factor weights

### Best public framing
Use language like:
- "Evidence carried forward"
- "Decision memory"
- "Contradiction detected"
- "Checkpoint overdue"
- "Route not yet earned"
- "Action not yet verified"
- "Pattern is recurring"
- "Options may be narrowing"
- "Cost of inaction estimated"

Avoid language like:
- "Our proprietary graph computes..."
- "Our kernel weights..."
- "Our arbiter tournament checks..."
- "The algorithm proves..."
- "AI determined..."

Surface the **effect**, not the machinery.

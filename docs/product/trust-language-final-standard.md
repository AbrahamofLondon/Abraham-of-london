# Trust Language Final Standard

**Date:** 2026-05-09
**Purpose:** Approved language bank for all public, client-facing, and operator-facing surfaces.

---

## USE — Approved Trust Language

### Product Identity
- "Decision Infrastructure by Abraham of London"
- "Decision Infrastructure"
- "A governed decision institution"

### System Capability
- "The system records, remembers, and follows up."
- "The system does not fabricate checkpoints, evidence, or outcomes."
- "The system does not invent certainty."
- "The system refuses to proceed when evidence is insufficient."
- "Nothing resets. Evidence carries forward."
- "Contradictions accumulate. The system does not forget."
- "The system can refuse to proceed."

### Evidence Posture
- "Based on your stated decision context and declared consequence."
- "Scenario only — not a financial forecast."
- "These consequences have not been independently verified."
- "Estimated exposure based on stated inputs."
- "This is an estimate, not a verified external fact."
- "Self-reported evidence is never represented as independently verified."
- "Source: {instrument} — Evidence posture: {user-reported|system-inferred|estimated}"
- "It does not claim a verified outcome unless evidence is actually provided."

### Earned Progression
- "Your evidence now supports a deeper report."
- "You can proceed, or keep the current checkpoint active."
- "Evidence threshold met."
- "Not yet earned by evidence."
- "Progression is earned by evidence, not pushed by the site."
- "Your current finding and checkpoint remain active."

### Counsel & Escalation
- "Counsel review is available when the evidence shows automated analysis should not proceed alone."
- "Counsel Review is not a starting point."
- "Counsel is warranted by the evidence."
- "Ongoing oversight is warranted by this evidence pattern."

### Governance & Trust
- "Every governed output can be challenged."
- "Challenges enter the case record."
- "No sale if the case is not ready."
- "Durable proof, not performance theatre."
- "This is a status surface, not a sales funnel."
- "Without pretending automation exists where it does not."

## AVOID — Weak or Risky Language

These terms should not be used on serious product surfaces. They may appear in content/editorial contexts where appropriate.

| Term | Why | Replacement |
|------|-----|-------------|
| "insights" | Soft, passive, generic | "findings", "signals", "decision record" |
| "clarity" (as marketing adjective) | Generic SaaS | "precision", "position confidence" |
| "solution" | Consulting language | "intervention", "directed action" |
| "dashboard" (for the product) | Passive data viz | "Decision Centre", "case console" |
| "tool" (for the product) | Diminishes institutional identity | "instrument", "system" |
| "recommendation" (from the system) | Advisory language | "directive", "governed output", "earned next step" |
| "transform" | Generic motivational | Describe specific mechanism |
| "data-driven" | Generic SaaS | "evidence-led", "governed" |

---

## FORBIDDEN — Must Never Appear on Public Surfaces

| Term | Why | Exception |
|------|-----|-----------|
| "AI-powered" | Frames product as AI wrapper | None |
| "AI-accelerated market baseline" | Unsubstantiated benchmark claim | None |
| "consulting" (as product identity) | Product is not a consultancy | OK in "not consulting" contrast |
| "advisory" (as product identity) | Product is not advisory | OK in bio/background context |
| "Institutional Platform" | Generic enterprise SaaS label | None — use "Decision Infrastructure" |
| "unlock" | SaaS paywall language | None — use "access" or "enter" |
| "Upgrade Now" | Purchase framing | None — use earned progression language |
| "upsell" / "funnel" | Sales language | OK only in explicit rejection ("not a funnel") |
| "book a call" / "request advisory" | Consulting engagement | None |
| "success rate" | Overclaim without evidence | None |
| "guarantee" (positive claim) | Unsubstantiated | OK in negative: "does not guarantee" |
| "verified" (for self-reported data) | Overclaim | OK only with proper evidence posture label |
| "proven" (without evidence chain) | Overclaim | OK only when evidence chain is explicit |
| "trusted by" (without evidence) | Unsubstantiated | Only with verified client consent |

## Context-Specific Replacements

| Context | Instead of... | Use... |
|---------|---------------|--------|
| Diagnostic result | "Your pattern shows..." | "The evidence you stated suggests..." |
| Financial projection | "Cost: £X" | "Estimated cost: £X — scenario only, not a financial forecast" |
| System inference | "The system determined..." | "The evidence supports..." |
| Counsel availability | "Book advisory support" | "Counsel is warranted by the evidence" |
| Boardroom access | "Upgrade to boardroom" | "Not yet qualified — evidence missing: [list]" |
| Product progression | "Recommended: Executive Reporting" | "Your evidence now supports Executive Reporting" |
| Locked feature | "Unlock Premium Access" | "Available when evidence threshold is met" |
| Outcome claim | "Verified outcome" | "User-reported outcome" (unless operator-confirmed) |
| Stopping | (no mention) | "Your current finding and checkpoint remain active" |

# Phase 6-14 Master Roadmap

Status: PLANNED
Live surface truth: This roadmap sequences work. It does not indicate that the later-phase operating system is already live.

## Objective

Sequence the institution-building work from commercial architecture through stewardship without colliding with the live implementation lane.

## Program Rule

Milestone A must close before Codex-originated planning turns into broad live-surface implementation.

Until then, Codex work remains:

- documentation
- standards
- architecture
- inventories
- copy and planning prep

| C | Authority and market position pack | Codex | Content-to-funnel, framework registry, distribution logic documented |
| D | Institutional architecture pack | Codex | Governance, continuity, stewardship, and final sequencing documented |

## Dependency and Risk Register

| Item | Dependency | Risk if early | Mitigation |
|---|---|---|---|
| Public copy renaming for Membership | Milestone A closure | Conflicts with live auth and route expectations | Hold to docs and inventories first |
| Diagnostics funnel copy cleanup | Claude closure | Can break route truth or event logic assumptions | Wait for live-lane checkpoint |
| Strategy Room structural changes | submit/enrol path stabilized | Could collide with active API convergence | Restrict to docs and operator packs |
| Tier vocabulary normalization in code | auth lane closure | Can create entitlement drift | Keep business-language standard separate from live code until safe |
| Authority CTA rollout | stable measurement foundation | Cannot judge impact without reliable funnel telemetry | Tie rollout to reporting spec |

## Safe Implementation Queue After Milestone A

1. Navigation and copy standardization using the naming matrix.
2. Non-critical link cleanup for Essays, Resources, Membership, and Vault labels.
3. Product page copy revisions for Executive Reporting and Strategy Room.
4. Membership and Vault value proposition clarification.
5. Controlled framework registry launch.

## Executive Summary

The system should be built in this order:

1. close the live operation
2. lock the commercial architecture
3. scale authority into market language
4. formalize governance and continuity
5. synthesize the institution into one operating system

That sequencing preserves speed without inviting preventable breakage.

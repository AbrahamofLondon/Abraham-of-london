# External Assurance Roadmap

**Status:** Internal planning document — not a public commitment  
**Last updated:** 2026-05-18

This document tracks the external assurance work that needs to happen, in rough priority order. Items here represent real work, not wording changes.

---

## Next 30 days

- [ ] Identify penetration test provider (CREST-accredited or equivalent)
- [ ] Define pen-test scope: web application, API surface, authentication flows, access control
- [ ] Prepare staging/test credentials and test account structure
- [ ] Define rules of engagement with provider
- [ ] Confirm budget approval for scoped engagement
- [ ] Document remediation process: how findings are classified, prioritised, and tracked
- [ ] Establish finding disclosure policy (what can be shared with prospects under NDA, what cannot)

---

## Next 60–90 days

- [ ] Complete scoped external penetration test
- [ ] Receive pen-test report
- [ ] Remediate critical and high findings
- [ ] Produce bounded attestation summary: "a scoped penetration test was completed on [date] covering [scope] by [provider]"
- [ ] Add pen-test readiness material to security assurance pack registry (update from "readiness" to "completed")
- [ ] Conduct SOC 2 readiness gap assessment if enterprise demand validates the investment
- [ ] Evaluate DPO appointment based on data processing volume and enterprise client requirements

---

## 90 days onwards (subject to enterprise demand and commercial justification)

- [ ] SOC 2 Type I readiness and observation
- [ ] SOC 2 Type II observation period
- [ ] Public status page with incident history feed
- [ ] Enterprise SSO / SAML integration (driven by enterprise client requirements)
- [ ] Enforced organisation-level MFA
- [ ] External WORM / blockchain anchoring for provenance records
- [ ] ISO 27001 organisational certification (if commercially justified)
- [ ] Formal disaster recovery testing with documented RTO/RPO
- [ ] Named DPO appointment or formal privacy governance programme

---

## What this roadmap does not claim

- This is a planning document, not a contractual commitment.
- "Planned" means intended, not guaranteed.
- Timeline is approximate and subject to commercial priorities.
- Completed items will be updated in the registry and assurance pack.
- Prospects should not be told items are completed until they actually are.

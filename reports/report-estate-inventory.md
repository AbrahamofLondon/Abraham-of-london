## Phase 0 Findings

- No universal `ArrivalScreen`, `report-experience-standard`, `report-output-registry`, `arrivalImplemented`, or `ReportOutputRegistry` equivalent exists.
- Sellability is currently governed by fulfilment readiness only; arrival quality is not a product gate.
- The strongest reusable PDF primitives are `BriefCoverPage` and `ForensicMarkLayer`, but they are not universal and do not replace a transmission page.
- The Boardroom Dossier web route is token-gated, but the access moment is a loading state rather than an arrival.
- The Boardroom Dossier PDF has a cover, but not a REAS v2 transmission page or 13-section architecture.
- Static PDFs and public download assets can bypass future report experience controls unless wrapped by the new output registry and checker.
- Bare Helvetica, single-page compression, absolute-position framing, and missing long-input smoke tests are the highest React-PDF stability risks.

## P0 Upgrade Set

The first build pass should prioritize:

1. `boardroom_brief` / `boardroom_dossier_web` / `boardroom_dossier_pdf`
2. `executive_reporting`
3. `oversight_brief`
4. `gmi_q1_2026`
5. A representative paid interactive instrument output

This creates the reusable pattern without attempting to redesign all static PDF assets in one pass.
